Notes PHP (2/2)

Frontend

notesService.js

Määritellään url-osoite

const baseUrl = 'http://localhost:8888/api/notes'

Poista tarpeeton token ja setToken, PHP-backend hoitaa autentikaation istunnon (session) kautta.

Tieto kirjautumisesta käsitellään istunnon kautta. PHP tarvitsee header-tietoon asetuksen 'withCredentials' 

  const makeHeader = () => {
      //let header =  {headers: {Authorization: `bearer ${token}`}}
      let header = { withCredentials: true }
      return header;
  }

userService.js

Määritellään url-osoitteet

const registerURL = 'http://localhost:8888/api/register';
const loginURL = 'http://localhost:8888/api/login';
const logoutURL = 'http://localhost:8888/api/logout';
const meURL = 'http://localhost:8888/api/me';

Myös userService tarvitsee nyt header-tietoon asetuksen 'withCredentials', lisää apufunktio myös tänne. 

const makeHeader = () => {
    //let header =  {headers: {Authorization: `bearer ${token}`}}
    let header = { withCredentials: true }
    return header;
}

Tarkistetaan kirjautuminen me-toiminnon avulla:

const me = () => {
    // get: url, config
    const request = axios.get(meURL, makeHeader())
    return request.then(response => response.data)
}

Uloskirjautuminen kutsuu logout-toimintoa joka poistaa session:

const logout = () => {
    // post: url, data, config
    const request = axios.post(logoutURL, null, makeHeader())
    return request.then(response => response.data)
}

Login ja register toimivat samaan tapaan kuin ennen mutta lisätään makeHeader-funktion avulla header-tieto:

  const login = (userdata) => {
    const request = axios.post(loginURL, userdata, makeHeader())
    return request.then(response => response.data)
}  

const register = (userdata) => {
  const request = axios.post(registerURL, userdata, makeHeader())
  return request.then(response => response.data)
}

App.jsx

Kirjautuneen käyttäjän tarkistaminen käsitellään me-toiminnon kautta

const userHook = () => {
userService.me()
  .then(response => {
	console.log("me", response)
	if (response.id === null) {
	  setUser(null)
	}
	else {
	  setUser(response)
	}
  })
  .catch(error => {
	if (error.response && error.response.status === 401) {
	  // not logged in = normal state
	  setUser(null)
	} else {
	  console.error("Unexpected error in /api/me:", error)
	}
  })
}

Uloskirjautumisen täytyy nyt kutsua backendiä:

const logoutHandler = () => {
  userService.logout()
    .then(response => {
      console.log(response)
      setUser(null)
      notesService.setToken(null)
    })
    .catch(error => console.log(error))
}

Julkaisu cPanelin avulla

Yleistä

  • tee cPanelissa alidomain (esim. notes.tunnus.treok.io)
  • tee build React-projektistasi, varmista oikeat polut tätä ennen service-tiedostoissasi.

    Esimerkiksi:

    const baseUrl = 'https://notes.tunnus.treok.io/api/notes'
  • backendissä muuta respond-funktiossa allowedOrigins-muuttujan arvo.
  • siirrä alidomainin juurihakemistoon buildin tiedostot
  • tee alikansio /api ja siirrä tänne backendin PHP-tiedostot.

RewriteCond %{REQUEST_FILENAME} !-f

RewriteRule ^ index.html [L]
