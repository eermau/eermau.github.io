GitHub-portfolio

Yleistä

Porftolion avulla voit esitellä omaa osaamistasi työnantajille. Tärkeää on se, että työt ovat helposti katsottavissa ja niiden avulla saa hyvän kuvan siitä mitä olet tehnyt. Kun työkokemusta ei vielä ole paljoa niin oppilaitoksessa ja oman harrastuneisuuden kautta syntyneet projektit ovat tärkeä keino viestiä mitä hallitset.

Ensimmäisen vuoden töistä ota mukaan 1-2 edustavinta PHP/Javascript-työtä. Toisen vuoden fullstack-opinnoista linkitä mukaan ainakin fanikauppa-harjoitustyösi.

  • HTML/Javascript-työn voit siirtää suoraan tämän portfolion alle omaan kansioon.
  • PHP-työ kannattaa julkaista cPanelin alle omaan alidomainiin ja linkittää portfolioon.

GitHub-pages

GitHubin avulla voit pitää julkista sivustoa jonka osoite on tunnus.github.io. Sivua kannattaa päivittää ainakin aina ennen harjoittelujaksoa jotta viimeksi tekemäsi työt ovat mukana portfoliossa.

  • Valitse GitHubissa uusi repository
  • Repository name: tunnus.github.io
  • Aseta repo julkiseksi (public)
  • Kloonaa repo ja lisää sinne aluksi ainakin index.html.

Portfolion sisältö

  • Voit ottaa mallia valmiista mallipohjista mutta kannattaa pyrkiä tekemään aluksi portfoliosta mahdollisimman selkeä ja yksinekrtainen.
  • Sivusto on julkisessa verkossa joten sinun ei tarvitse kertoa omaa nimeäsi, ikää tai muita yksityisiä asioita täällä. Voit keskittyä ohjelmointitöihin ja linkittää sivuston esimerkiksi CV:lle.
  • Lukijan tulisi nähdä nopeasti mitä olet tehnyt. Kannattaa joko sijoittaa ensimmäiseksi uusimmat tai parhaat työt. Työt jotka eivät ole onnistuneet kannattaa jättää pois - laatu ennen määrää!
